# 文档本地化规范

英文版： [documentation-localization.md](documentation-localization.md)

## 目的

这份文档规定 RoboCode 如何同时维护英文和简体中文文档。

## 当前规范

- 英文仍然是技术结构和文件命名上的 canonical base version。
- 简体中文作为一等配套文档维护，而不是可有可无的附加翻译。
- 所有长期保留的产品、架构、路线图、需求和计划类文档，都应有对应的 `*.zh-CN.md` 文件。

## 文件命名

- 英文源文件：`name.md`
- 简体中文对应文件：`name.zh-CN.md`

示例：

- `README.md`
- `README.zh-CN.md`
- `docs/architecture.md`
- `docs/architecture.zh-CN.md`

## 链接规则

- 每份英文文档都应在开头附近链接到中文版本。
- 每份中文文档都应在开头附近链接回英文版本。
- 像 `README.md` 这样的高层入口文档，应同时列出两种语言版本。

## 更新规则

- 当 canonical 英文文档发生实质性变化时，应尽量在同一个变更集中同步更新中文文档。
- 如果对应中文版本不能立刻同步更新，必须在同一个 PR 或 commit 描述中明确标记，而不是默认让两边长期漂移。
- 除非文件本身就是短小索引型文档，否则不要把中英文正文混写在同一份文件里。

## 适用范围

本规范适用于：

- 仓库总览文档
- 架构文档
- 产品需求
- 分阶段路线图
- gap matrix
- 计划长期复用的实现计划

本规范不要求为以下内容做双语复制：

- 生成产物
- 临时草稿
- 一次性的本地调试记录

## 风格建议

- 中英文版本尽量保持一致的章节结构。
- 文件名、命令名、枚举名和代码标识符保留原始形式。
- 翻译 prose，不翻译 code。
- 优先使用清晰的工程语言，而不是过度逐字直译。
